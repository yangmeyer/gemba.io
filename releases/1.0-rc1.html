Asset Catalog how-to

Add an @3x asset to an existing image set (that already contains corresponding @1x or @2x assets) by dropping it into Gemba. The image set’s Contents.json will be updated automatically, and it will appear alongside the new @3x asset in the asset list.

Asset Catalog limitations

  • Gemba expects the traditional naming convention: “icon[@3x][~ipad].png”
  • Not supported yet: Adding new image sets to an asset catalog.
  • No plans to support: “Launch Image” sets, creating new asset catalogs, asset slicing, size classes, device-specific “568h” assets.
